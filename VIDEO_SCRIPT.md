【开场 0:00-0:30】

大家好，我是 Yang。今天给大家介绍一个我用 Python 开发的小项目——百度AI通用图像识别器。

这是一个结合百度AI开放平台API的图像识别应用，支持命令行和图形界面两种使用方式。项目展示了如何安全地调用第三方API、处理环保变量、以及构建用户友好的界面。

═══════════════════════════════════════════════════════════

【项目背景 0:30-1:00】

为什么做这个项目呢？主要有三个原因：

第一，学习如何集成第三方API。百度AI开放平台提供了强大的图像识别能力，我们可以识别超过10万种物体和场景。

第二，安全性考虑。API Key 和 Secret Key 是敏感信息，不能硬编码在代码里。我采用环境变量的方式来管理这些凭证，这样代码上传到GitHub时完全不会泄露密钥。

第三，用户体验。我不仅开发了命令行版本，还用 tkinter 做了一个图形界面，让非技术用户也能轻松使用。

═══════════════════════════════════════════════════════════

【技术架构 1:00-2:00】

让我们看一下项目的核心代码结构。

【展示 main.py】

这是命令行版本。主要有四个函数：

第一个是 `get_access_token()`。它从环境变量读取 API Key 和 Secret Key，然后调用百度的 OAuth 接口获取 access_token。注意这里我用 `os.environ.get()` 来读取环境变量，如果没有设置会抛出 EnvironmentError，提示用户如何配置。

第二个是 `image_to_base64()`。图片需要转成 base64 编码才能通过 HTTP 请求发送给百度API。这个函数很简单，就是读取本地图片文件，用 base64 编码。

第三个是 `recognize_image()`。这是核心函数，它调用百度的通用物体和场景识别API。我们把 base64 编码的图片作为请求体发送，API 返回识别结果，包括标签名称和置信度分数。

第四个是 `print_results()`。这个函数格式化打印识别结果。我用了一个简单的进度条来可视化置信度——置信度越高，进度条越满。

【展示 app.py】

这是图形界面版本，用 tkinter 开发。界面分为三部分：

左侧是图片预览区。用户可以点击"选择图片"按钮，从本地选择一张图片，程序会自动显示缩略图预览。

右侧是识别结果表格。用 Treeview 组件展示识别结果，包括排名、标签名称、置信度百分比，以及一个可视化的进度条。

底部是状态栏，显示当前操作状态和进度。

关键的是，识别操作在后台线程中执行，这样界面不会卡顿。用户点击"开始识别"后，程序会启动一个线程来调用API，同时显示加载动画。

【环境配置 2:00-2:45】

现在演示一下如何在 PyCharm 里运行这个项目。

【打开 PyCharm】

首先，安装依赖。打开终端，运行 `pip install -r requirements.txt`。项目只需要两个依赖：requests 用来发送HTTP请求，Pillow 用来处理图片。

【展示 requirements.txt】

然后，最关键的一步——配置环境变量。

【打开 Run → Edit Configurations】

在 PyCharm 的运行配置里，找到 Environment variables 这一项。添加两个变量：

BAIDU_API_KEY = 你的 API Key
BAIDU_SECRET_KEY = 你的 Secret Key

这样做的好处是，环境变量只在 PyCharm 运行时生效，不会污染系统全局环境，也不会被意外提交到代码仓库。

【展示 .gitignore】

我们的 .gitignore 文件已经配置好了，会忽略 .env 文件。这样即使有人不小心创建了 .env 文件，也不会被上传到 GitHub。
